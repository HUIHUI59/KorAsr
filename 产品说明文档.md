# KorASR — 留学生课堂实时韩语同传助手
## 产品说明文档 v1.0

**文档日期**：2026-04-06  
**目标用户**：在韩国留学、以中文为母语的学生  
**运行环境**：本地 4090 显卡作为服务器，浏览器作为客户端  

---

## 一、产品定位

> "上课听不懂韩语？开着它，中文字幕实时出来。"

KorASR 是一款运行在本地 GPU 服务器上的**实时韩中双语字幕 + 课堂笔记**工具。它不依赖任何付费云端 ASR 服务，利用本地 4090 显卡跑 Whisper Large-v3 模型实现高精度韩语识别，再通过 AI 翻译成中文，全程低延迟呈现在浏览器页面。

---

## 二、核心使用场景

| 场景 | 描述 |
|------|------|
| **课堂听讲** | 笔记本放桌上，页面显示教授韩语 → 中文实时字幕 |
| **与教授会议** | 区分多个说话人，记录谁说了什么 |
| **课后复习** | 导出一整节课的双语文字记录，做成笔记 |
| **录音补课** | 上传录音文件，事后转写 + 翻译 |

## 三、功能清单

### 3.1 核心功能（已有基础，需优化）

---

### 3.2 会话管理（P0，最高优先级）

**为什么重要**：每次上课都是独立的知识单元，需要能保存、命名、再次查阅。

| 功能 | 描述 |
|------|------|
| 新建会话 | 开始录制前填写：课程名 + 日期，自动生成会话ID |
| 自动保存 | 实时将转写结果写入本地 JSON 文件，防止意外丢失 |
| 会话列表 | 首页展示历史会话，按日期 / 课程名搜索 |
| 继续会话 | 可以续接上一次未完成的录制 |
| 删除会话 | 删除指定历史记录 |

---

### 3.3 导出功能（P0）

课后需要把字幕导出成可编辑的笔记。

| 导出格式 | 内容 |
|----------|------|
| **Markdown** | 带时间戳的双语对照，可粘贴进 Notion/Obsidian |
| **纯文本 TXT** | 只有中文翻译，方便快速阅读 |
| **JSON** | 完整结构化数据，含时间戳、韩文、中文 |
| **剪贴板复制** | 单行复制（点击任意字幕卡片即可复制中文） |

导出内容示例（Markdown）：
```
## 有机化学 — 2026-04-06

**[00:03:12]**  
🇰🇷 오늘은 알케인의 구조에 대해 설명하겠습니다.  
🇨🇳 今天我来讲解烷烃的结构。

**[00:03:45]**  
🇰🇷 탄소 원자는 네 개의 공유 결합을 형성합니다.  
🇨🇳 碳原子可以形成四个共价键。
```

### 3.4 说话人识别 / 分离（P1）

**场景**：与教授开会时，需要区分"谁说了什么"。

- 集成 **WhisperX + pyannote** 说话人分离
- 自动检测麦克风中的说话人数量（无需预先设定）
- 为每位说话人分配颜色标签（教授 / 学生 / 自己）
- 可手动给说话人贴上名字标签（如"金教授"、"我"）

> **注意**：说话人分离会增加约 0.5-1 秒延迟，建议仅在"会议模式"下开启。

---

### 3.5 AI 课堂总结（P1）

课程结束后，一键生成结构化摘要。

摘要内容包括：
- **核心主题**：这节课讲了什么
- **关键术语**：专业词汇列表（韩文 + 中文）
- **重点内容**：5-10 条核心知识点
- **疑问点**：自动识别疑问句（可能是教授留的思考题）
- **下节预告**：识别教授提到的"下次课"内容

生成方式：将整节课转写内容发给 Moonshot API，用专门的 Prompt 生成学术摘要。

---

### 3.6 关键词高亮 & 专业词库（P1）

| 功能 | 描述 |
|------|------|
| 自动关键词提取 | 每个句子标注核心术语（黄色高亮） |
| 个人词库 | 按课程建立专属词库，下次上课时自动高亮已知术语 |
| 术语纠错 | 用户可标记错误翻译的术语，系统记录并优化 |

---

### 3.7 笔记模式（P2）

在字幕区域旁开放"笔记栏"，允许：
- 对任意一句字幕添加文字注释
- 标记"重要"（⭐ 书签），后续可过滤只看重要内容
- 手写板截图插入（手机端）

---

### 3.8 录音文件处理（P2）

不只是实时转写，还支持上传已有录音：
- 支持格式：MP3 / WAV / M4A / OGG
- 服务器端批量处理，速度约为实时速度的 10-20 倍
- 处理完成后自动生成会话，和实时会话统一管理

---

### 3.9 移动端适配（P2）

有时在教室里用手机比电脑方便：
- 响应式 UI，手机浏览器可用
- 手机麦克风支持（需 HTTPS，建议配 Tailscale / ngrok）
- 字体放大，适合在嘈杂环境下快速扫视

## 四、技术架构

```
┌─────────────────────────────────────────────────────────┐
│                      客户端（浏览器）                     │
│  HTML/JS → 麦克风采样 → 16kHz float32 → WebSocket 推送  │
└────────────────────────┬────────────────────────────────┘
                         │ WebSocket
┌────────────────────────▼────────────────────────────────┐
│                  本地服务器（4090）                        │
│                                                          │
│  FastAPI + Uvicorn                                       │
│      │                                                   │
│      ├── VAD 静音检测（自制 RMS）                         │
│      ├── Faster-Whisper Large-v3（CUDA fp16）            │
│      │   └── 韩语 ASR，beam_size=5，上下文记忆             │
│      ├── [可选] WhisperX + pyannote 说话人分离            │
│      │                                                   │
│      ├── Moonshot API（韩 → 中 翻译）                    │
│      ├── Moonshot API（课堂总结）                         │
│      │                                                   │
│      └── 本地 SQLite（会话存储）                          │
└─────────────────────────────────────────────────────────┘
```

### 关键技术选型

| 模块 | 技术 | 理由 |
|------|------|------|
| ASR | Faster-Whisper Large-v3 | 精度最高，4090 上延迟 < 1s |
| 翻译 | Moonshot API（moonshot-v1-8k） | 中文表达自然，成本低 |
| 说话人分离 | WhisperX + pyannote 3.1 | 开源最佳方案 |
| 课堂总结 | Moonshot API（moonshot-v1-32k） | 长文本处理 |
| 存储 | SQLite（本地） | 零配置，隐私安全 |
| 后端 | FastAPI + WebSocket | 已有基础，异步友好 |
| 前端 | 原生 HTML/JS | 无需构建，轻量 |

## 五、隐私与安全

| 项目 | 方案 |
|------|------|
| 录音数据 | 全部本地处理，不上传任何音频到云端 |
| 转写文本 | 仅翻译内容发送给 Moonshot API |
| API Key | 改为环境变量配置，不写在代码里 |
| 外网访问 | 推荐使用 Tailscale（点对点 VPN）而非公网暴露 |
