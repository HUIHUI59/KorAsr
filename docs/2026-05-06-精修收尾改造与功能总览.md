# KorASR 开发报告 — 2026-05-06

主题：**会话结束时的收尾流程**。
上一阶段把实时 ASR + 60s LLM 精修两条管道跑通了，但会话结束瞬间未到 60s 节拍的那段内容存在丢失或走低质量路径的问题。本次补齐这块，并顺手收紧几个相关问题，同时调整前端字号。

## 一、本次改动

### 1.1 修复的问题

| # | 问题 | 影响 |
|---|---|---|
| ① | 尾段精修写了 DB 但 WS 已断开未推前端 | 最后一段精修翻译看不到 |
| ② | 尾段精修走低质量路径（拼 `segments.ko_text`），不重做长稿 ASR | 精修退化为实时 ASR |
| ③ | 用户停录时 `Transcriber.audio_buffer` 残留 1–5s 未 commit 被丢弃 | 最后一两句没 segment 也没 polish |
| ④ | `_polish_loop` 单次异常停掉所有后续 60s 节拍 | 一次失败 → 整个 polish 管道死透 |
| ⑤ | 前端中文字号偏小（13/14 px）、描述文字冗余 | 阅读累 |

### 1.2 后端（`backend/ws/handler.py` 重构）

- **WS 协议引入 graceful stop**：主循环切到 `websocket.receive()`，同时接 PCM bytes 与 JSON 文本控制帧。前端 stop 时不再直接 `ws.close()`，而是发 `{"action":"stop"}`；后端跑完整收尾流程后由 backend 主动 close。
- **新 `_graceful_finalize()` 三步收尾**
  1. `Transcriber` 残留 buffer ≥ 0.5s → `commit(was_forced=False)` 全部吐出 → 跑 final ASR/翻译 → 推 done 帧 + 写 segment（修 ③）
  2. flush PCM 文件
  3. 从 `polish_last_pcm_byte` 读到 EOF → `transcribe_long_form` + `polish_translate` → 推 polish 帧（修 ① ②）
- **`_do_polish_iteration(send_ws)` 抽出**：60s 节拍 loop / graceful 收尾 / abrupt 断开三处复用。abrupt 断开也调一次 `send_ws=False`，DB 落盘以便 history 查得到。
- **`_polish_loop` 逐次 try/except**：单次失败（MLX OOM / Moonshot 429 等）只 log，loop 继续（修 ④）。
- **精修管道状态提升到外层作用域**：`polish_last_pcm_byte` / `polish_chunk_index` / `polish_last_processed_seq` 不再是 `_polish_loop` 局部变量，三条收尾路径（节拍 / graceful / abrupt）都能正确推进。

## 二、当前已实现功能

### 2.1 ASR
- 双后端：**MLX whisper-large-v3-turbo**（Apple Silicon GPU/Metal）/ **faster-whisper**（CTranslate2，CPU/CUDA），`ASR_BACKEND` env 切换。
- **Silero VAD**：min 0.5s / max 8s / 句尾静音 600ms 触发。
- **智能切点**：撞 8s 上限时在最后 1 秒里 100ms 滑窗找最低 RMS 处切，避免词中央切。
- **4 层幻觉过滤**：低 RMS 门、prompt-echo 检测（最长子串 ≥70% / SequenceMatcher ≥0.65）、重复循环检测（distinct/total < 30%）、低能量 + 常见短语精确匹配（如 `감사합니다` / `네` 在 RMS<0.025 时屏蔽）。
- **Interim/Final 共享 GPU semaphore**：interim 检测到 GPU 忙立即跳过，final 一定排队。

### 2.2 翻译
- **实时**：Moonshot `moonshot-v1-auto`（自动路由抗 `v1-8k` 拥堵）+ SSE 流式 + 100ms 节流 WS 推送。
- **精修**：60s 节拍 + graceful 收尾 + abrupt 兜底；long-form Whisper（`condition_on_previous_text=True`，由 Whisper 自己分句）+ LLM 整稿翻译，术语跨段一致。

### 2.3 存储
- SQLite + SQLModel：`Session` / `Segment` / `PolishedChunk`。
- 原始 PCM 存盘 `data/raw/<sid>.pcm` —— 精修管道的真源数据。
- 课堂 Notes 持久化（节流 1s 写盘）。

### 2.4 前端（Vue 3 + Pinia）
- 三态滚动卡片：interim（橙）/ translating（紫）/ done。
- ✨ 精修翻译面板（折叠看原韩文）。
- 桌面 3 栏（Live + Polish + Notes）/ 移动 3 tab 自适应。
- HTTPS 自签证书 → 局域网 / 手机直接打开（mic 权限通过）。

### 2.5 工程
- 多 session 共享单 GPU semaphore，避免推理冲突。
- 启动脚本自动生成自签证书（覆盖本机所有 IP）。
- 一键启动 `./startAll.sh`。
